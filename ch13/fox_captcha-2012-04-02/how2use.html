Introduction Version 3.0

Welcome to Fox_captcha class. The class is meant by creating Captcha verification to secure your forms from floods and spamers. The class is easy to use. It is just two steps to do in the form page and another one step in the form's action page to get Captcha working (of course after the ordinary class's object initiation steps).

The version 2 of Fox_captcha that has modifications on the way by which the reloading of captcha image is carried out. The modification was made due to the problems related with the image refreshment regarded in the discussion forum of Fox_captcha and very much thanks to Michele Castellucci. I did not use AJAX, It is only another dynamic image file and some javascript code to allow the reload functionality working properly.

version 2.5 is released and it has one bug fix and one new feature:

  • The bug fixed: It is a bug related with number of lines rendered via $lines_amount property. By default, it should renders 5 lines, when you defined a value to this property it should generates exact number of lines as defined.
  • The new feature: While loading the new Captcha Image, a loading gif indicator will be rendered till the new Captcha Image is completely downloaded. The file name of the loading indicator gif must be " fox_captcha_image_loading.gif " and it must be available at the same directory of $class_dir property.

    Hint: You could able to use a free online service called Chimply to generate the laoding gif indicator. We suggest to set the indicator image height to be equals to the $img_height property which it was set during the object initiation.

Currently version 3 is released and it has two changes:

  • The image becomes totally dynamic: it means that you don't have to make special writing permission to the directory that handles the class. i.e you will not see fox_captcha_image.jpg any more.
  • the $class_dir property is renamed: We found that the name of the property is not descriptive, so we changed it to $image_dir. It will catch the directory path in-which both fox_captcha_image.php and fox_captcha_image_loading.gif are resident. So you MUST change this property name, if you had used it before, to use version 3 correctly
